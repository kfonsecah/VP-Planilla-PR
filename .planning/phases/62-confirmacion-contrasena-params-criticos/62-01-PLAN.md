# Phase 62-01: Backend Security & Audit - Plan

## Wave 1: Backend Implementation
**Goal:** Enforce password re-confirmation for critical legal parameters and log the confirmation status in audit logs.

### Task 1: AuthService Extension
- **File:** `src/backend/src/service/AuthService.ts`
- **Action:** Add `verifyPasswordForUser(userId: number, plainPassword: string): Promise<boolean>` method.
- **Read first:** `src/backend/src/service/AuthService.ts`
- **Acceptance criteria:**
  - Method exists and is public.
  - Correctly compares plainPassword with hashed password from DB.
  - Returns true for correct password, false otherwise.
  - No plain-text password logging.

### Task 2: Controller Enforcement
- **File:** `src/backend/src/controller/LegalParamController.ts`
- **Action:** 
  - Update `upsertParam` and `patchParam`.
  - Fetch existing param or check `isCritical` in payload.
  - If critical: require `confirmationPassword` in body.
  - Call `AuthService.verifyPasswordForUser`.
  - Return 400 if missing, 403 if incorrect.
- **Read first:** `src/backend/src/controller/LegalParamController.ts`
- **Acceptance criteria:**
  - Critical updates without password fail with 400.
  - Incorrect password fails with 403.
  - Success only with correct password.

### Task 3: Audit Logging Integration
- **File:** `src/backend/src/service/LegalParamService.ts`
- **Action:** Update `upsertParam` to use `prisma.$transaction`.
- **Details:** 
  - Wrap the param update and `vpg_audit_logs.create` in a single transaction.
  - Include `action: 'LEGAL_PARAM_UPDATE'`, `password_confirmed: true` in metadata.
- **Acceptance criteria:**
  - `vpg_audit_logs` table receives a new entry on success.
  - Failure in any step rolls back the entire operation.
  - Entry contains old/new values and confirmation flag.

---

## Verification Criteria (Nyquist D8)
1. **Security:** Attempting to patch `OT_FACTOR` (critical) via Postman without `confirmationPassword` returns 400.
2. **Persistence:** Successful change with password generates `password_confirmed: true` in `vpg_audit_logs`.
