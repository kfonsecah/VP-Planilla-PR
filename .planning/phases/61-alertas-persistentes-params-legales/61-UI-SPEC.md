## Component Inventory

### 1. LegalParamAlertBanner (new)

Location: `src/frontend/src/components/LegalParamAlertBanner.tsx`

**Trigger:** Renders above the stats grid in `main/page.tsx` when `GET /notifications?type=LEGAL_PARAM_CHANGE&unacknowledged=true` returns one or more results (admin or payroll_manager roles only).

**Layout:**
```
[ShieldExclamationIcon 20px] [Title: "Parámetro legal modificado"] [collapse chevron] [×]
                              [Expanded: param name, old → new, who, when]
                              [If multiple: "N alertas sin revisar — ver todas"]
                              [CTA: "Marcar como revisado" — admin only]
```

**States:**
- Single alert, expanded by default: shows full message + CTA
- Multiple alerts: shows first one expanded, secondary ones collapsed with count badge (`+N más`)
- All acknowledged: component unmounts (AnimatePresence exit)
- Loading: skeleton pulse (same pattern as `NotificationPanel.tsx` lines 98–110)

**Behavior:**
- Collapse/expand: chevron toggle, no page navigation
- "Marcar como revisado": calls `PATCH /notifications/:id/acknowledge`, then refetches; button shows spinner during request
- Dismiss (×): hides locally for the session only — does NOT mark as acknowledged
- Admin-only: `acknowledged_by` button hidden for `payroll_manager` role

**Animation:** `AnimatePresence` + `motion.div` with `backdropVariants` / `modalVariants` (same pattern as `NotificationPanel.tsx` lines 22–31). Entry: `opacity 0→1, y -8→0` in 150ms. Exit: `opacity 1→0, y 0→-8` in 100ms.

### 2. LegalParamAlertBadge (new — inline in configuracion/empresa)

Location: inline JSX in `src/frontend/src/app/pages/configuracion/empresa/page.tsx`

**Trigger:** Renders next to the label of any `FEATURE_FLAG` parameter that has `value = 0` (MIN_WAGE_CHECK_ENABLED, etc.).

**Visual:**
```
<span class="bg-red-100 dark:bg-red-900/30 text-red-700 dark:text-red-400 text-[10px] font-semibold uppercase px-1.5 py-0.5 rounded-full">
  DESACTIVADO — Riesgo legal
</span>
```

No animation. Renders conditionally when parameter value is falsy.

---

### 3. NotificationPanel — LEGAL_PARAM_CHANGE differentiation (extend existing)

Location: `src/frontend/src/components/ui/NotificationPanel.tsx`

**Change:** When `notification.notification_type === 'LEGAL_PARAM_CHANGE'`, render a `ShieldExclamationIcon` (20px, `text-red-500`) instead of the generic blue dot. The panel row background uses `bg-red-50/50 dark:bg-red-900/10` instead of `bg-[#FFF9E6]`.

**Bell badge in Header:** When any unacknowledged `LEGAL_PARAM_CHANGE` exists, show a second indicator — a 6px solid red dot at bottom-right of the bell icon (in addition to the unread count badge at top-right). This signals legal urgency independently of general unread count.

---

### 4. PayrollWizard Step 1 — Legal param warning banner (new)

Location: `src/frontend/src/components/PayrollWizard.tsx` (injected before period selection cards)

**Trigger:** `GET /notifications?type=LEGAL_PARAM_CHANGE&unacknowledged=true` returns alerts with `acknowledged_at` newer than the current payroll's `created_at`.

**Visual:** Amber banner (matches existing `main/page.tsx` amber pattern):
```
[ExclamationTriangleIcon] "Los parámetros legales cambiaron desde que esta planilla fue creada."
                           "Se recomienda recalcular antes de aprobar."
```

No CTA — informational only. Dismissible with × (session-only).

## Interaction States

### LegalParamAlertBanner

| State | Visual |
|-------|--------|
| Loading (initial fetch) | 3-row skeleton pulse — matches NotificationPanel skeleton |
| No alerts | Component absent (no render) |
| 1 alert — no risk | Blue banner, `InformationCircleIcon`, collapsed by default on mobile, expanded on desktop |
| 1 alert — legal risk | Red banner, `ShieldExclamationIcon`, always expanded |
| Multiple alerts | Red banner (if any has risk), first shown, others collapsed with `+N más` |
| Acknowledging (in progress) | "Marcar como revisado" shows `ArrowPathIcon` spinner + disabled |
| Acknowledged | Banner exits via AnimatePresence — 100ms fade+slide |
| Dismissed locally | Banner exits immediately, no API call |

### DESACTIVADO badge

| Condition | Renders |
|-----------|---------|
| `value === 0` on FEATURE_FLAG param | Badge visible next to label |
| `value !== 0` | Badge absent |
