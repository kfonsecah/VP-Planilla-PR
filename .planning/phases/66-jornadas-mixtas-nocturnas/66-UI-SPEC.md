## Component Specifications

### 1. Shift Type Dropdown (Employee Form)

**Location:** 
- Create Employee Modal (`AddEmployeeModal.tsx`)
- Edit Employee Modal (`EditEmployeeModal.tsx`)
- Position: Below employee name, above hourly rate field

**Field properties:**
- Label: "Jornada de trabajo" (16px, bold, verde-primary)
- Label note (optional): "(Horario estándar por día de 8h, 7h mixta, o 6h nocturna)"
- Field type: HTML `<select>` or custom dropdown component
- Width: Full width of form (constrained to form max-width)
- Background: white dark:zinc-900
- Border: 1px border-zinc-300 dark:border-zinc-700
- Border radius: rounded-md (6px)
- Padding: p-sm (8px) horizontal, p-xs (4px) vertical for comfort
- Font size: 14px (body)

**Options (4 total):**

```
1. Label: "Default de empresa (actualmente: [ShiftType actual de enterprise])"
   Value: "USE_ENTERPRISE_DEFAULT"
   Description: "Usa la configuración estándar de la empresa"
   
2. Label: "Jornada diurna (8h/día)"
   Value: "DIURNA"
   Description: "Jornada estándar: 8 horas laborales por día"
   
3. Label: "Jornada mixta (7h/día)"
   Value: "MIXTA"
   Description: "Jornada mixta: 7 horas laborales por día"
   
4. Label: "Jornada nocturna (6h/día)"
   Value: "NOCTURNA"
   Description: "Jornada nocturna: 6 horas laborales por día"
```

**Option styling:**
- Option text: 14px, regular weight
- Option description (if visible in custom dropdown): 12px, verde-secondary, regular weight
- Hover state: bg-verde-card (#FCF1D5) / dark:bg-zinc-800
- Selected state: bg-verde-primary text-white, bold weight
- Disabled state (if applicable): opacity-50, cursor-not-allowed

**Default value:**
- New employee: `USE_ENTERPRISE_DEFAULT` (inherits enterprise setting)
- Edit employee: Load from `vpg_employees.shift_type` field

**Validation:**
- Required field (cannot be null after save)
- No custom validation — all 4 enum values are valid
- Type: Zod enum validation in form schema

**Dark mode:**
- Label text: text-zinc-800 dark:text-white
- Field: bg-white dark:bg-zinc-900
- Border: border-zinc-300 dark:border-zinc-700
- Options: white dark:zinc-800 background

## Interaction Specs

### Employee Form — Shift Type Dropdown

**Behavior:**
1. Field loads with `USE_ENTERPRISE_DEFAULT` for new employees
2. Field loads with current employee's `shift_type` value for edits
3. User selects from 4 options
4. No validation errors (all options valid)
5. On save: Form POSTs to backend with `shift_type: string` in request body
6. No client-side confirmation dialog needed (shift type is non-critical)

**React Hook Form integration:**
- Field name: `employee_shift_type` (follows convention: `entity_field_name`)
- Zod schema: `z.enum(['USE_ENTERPRISE_DEFAULT', 'DIURNA', 'MIXTA', 'NOCTURNA'])`
- Default value: `USE_ENTERPRISE_DEFAULT` (new) or loaded from DB (edit)
- Register: `register('employee_shift_type')`
- No async validation needed

**Edge cases:**
- Enterprise default changes after employee is created: Employee's explicit shift_type overrides it (no automatic sync)
- Switching from explicit to `USE_ENTERPRISE_DEFAULT`: No data loss (explicit value can be overridden again)

### Wizard Step 3 — Tooltip on Hover

**Behavior:**
1. Wizard loads employee data with `shift_type` field populated
2. Row renders with employee name
3. If `shift_type !== 'USE_ENTERPRISE_DEFAULT'`:
   - Hover over name cell triggers tooltip
   - Tooltip shows shift name and daily hour cap
   - Tooltip disappears on mouse leave (300ms delay)
4. If `shift_type === 'USE_ENTERPRISE_DEFAULT'`:
   - No tooltip shows (row behavior unchanged from before Phase 66)

**Accessibility:**
- Tooltip should be keyboard-accessible via focus (if icon present)
- Screen readers: Use `aria-label` on icon: "Jornada: [ShiftType] ([Hours]h/día)"
- Tab order: Icon is focusable if present, tooltip appears on focus

**Mobile behavior:**
- Touch devices: Tap icon to toggle tooltip visibility (not hover)
- On tablets: Long-press name cell shows tooltip
- Dismiss: Tap elsewhere or swipe

## API Integration Points

**Frontend calls backend via `@/services/http.ts` ONLY:**

1. **POST /employees** (new employee)
   - Request includes: `employee_shift_type: 'DIURNA' | 'MIXTA' | 'NOCTURNA' | 'USE_ENTERPRISE_DEFAULT'`
   - No validation errors — all values accepted
   - Response: Created employee object with `shift_type` echoed back

2. **PATCH /employees/:id** (edit employee)
   - Request includes: `employee_shift_type: enum`
   - Response: Updated employee object with `shift_type` field

3. **GET /payroll/:id** (wizard step 3 load)
   - Response includes: Array of employees with `shift_type` field populated
   - Used to determine if tooltip should appear

**No new endpoints required.** Existing endpoints are extended to include `shift_type` field.
