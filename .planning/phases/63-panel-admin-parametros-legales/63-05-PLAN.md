## Contexto
Implementar la visualización y edición específica para parámetros de tipo booleano o toggles de sistema. Aborda además el manejo estricto de valores booleanos y el control de accesos (`readOnly`) advertido en la revisión.

<action>
Crear `src/frontend/src/components/FeatureFlagToggle.tsx`.
- Props: `param: LegalParam`, `onChange: () => void`, `readOnly?: boolean`.
- Es una tarjeta con el layout similar a `LegalParamCard` pero muestra un control Toggle (Switch animado).
- Parsear el valor de forma estricta para evitar falsos positivos: `const isOn = String(param.value) === '1' || String(param.value).toLowerCase() === 'true';`
- Si está en "OFF", mostrar un badge rojo con el texto "DESACTIVADO".
- **Enforcement `readOnly`**: Si `readOnly === true`, deshabilitar la interacción del Toggle y mostrar estilo visual deshabilitado (`opacity-50`, cursor-not-allowed).
- Si no es `readOnly` y se interactúa, y `isCritical === true`, disparar el modal de confirmación de contraseña. Al confirmar (o directamente si no es crítico), llamar a `patchParam` (enviando "1" o "0").
</action>
