**DailyRow.tsx**

Renders one EffectiveClockLog (one day's IN/OUT pair). Shows:
1. Date header (Spanish full format per UX-01)
2. IN mark row (or missing-mark alert)
3. OUT mark row (or missing-mark alert)
4. Source traceability icon next to each timestamp (D-09)
5. "Corregir" stub button for non-valid marks (Phase 35 deferred — disabled, no onClick handler)
6. "Ver detalles" stub for valid marks (disabled)

Props interface:
```typescript
interface DailyRowProps {
  log: EffectiveClockLog;
  onAddMissing?: (type: 'in' | 'out') => void;  // stub — no-op in Phase 34
  onCorrect?: () => void;  // stub — no-op in Phase 34
}
```

**Source traceability icon helper (D-09):**
Define inside the file as a small helper component or function:
- `java_import` or `device` source → clock icon (use `⏱` or SVG clock Heroicon)
- `manual` source → hand icon (`✋` or SVG HandRaisedIcon)
- `corrected` status on original → refresh icon (`🔄` or SVG ArrowPathIcon)

Render the icon as a small `title`-attributed `<span>` with tooltip text:
- clock: `title="Importado desde reloj de asistencia"`
- hand: `title="Entrada manual"`
- refresh: `title="Marca corregida/ajustada"`

Use `text-xs` spans for icons.

**Date format (UX-01):**
```typescript
const dateLabel = new Date(log.log_date + 'T00:00:00').toLocaleDateString('es-CR', {
  weekday: 'long',
  year: 'numeric',
  month: 'long',
  day: 'numeric',
});
```
Note: append `T00:00:00` to avoid UTC offset shifting the date.

**Effective time resolution:**
```typescript
const inTime = log.adjusted?.in_time ?? log.original.in_time;
```
