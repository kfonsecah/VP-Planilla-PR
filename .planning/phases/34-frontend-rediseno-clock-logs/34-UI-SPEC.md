|------|------|--------|-------------|-------------|
| Body text | 14px (text-sm) | 400 (regular) | 1.5 | PraderaFont / Inter fallback |

**Font weights:** 2 declared only — 400 (regular) for body text, 600 (semibold) for all emphasis (labels, headings, display).

- Labels/uppercase: `text-xs` with `font-semibold`

**Font behavior:** VerdeFont for customer-facing titles (page heading, branch names, employee names). PraderaFont for operational text (timestamps, status labels, help text).

## Layout & Interaction Patterns

### Hierarchy
The page layout follows this structure:

```
[Page Header]
  ├─ Page title + breadcrumb
  └─ Descriptive subtitle + date preset buttons

[Date & Filter Bar]
  ├─ Biweekly preset buttons: [1ra Quincena] [2da Quincena] [Mes Actual]
  ├─ Custom date range: [Desde: dd/mm/yy] [Hasta: dd/mm/yy]
  └─ Filter toggles: Status (valid/anomaly/orphan/pending/corrected)

[Summary Stats] (if marks exist)
  ├─ Card: X Válidas (green border-left)
  ├─ Card: Y Anomalías (amber border-left)
  ├─ Card: Z Huérfanas (red border-left)
  └─ Card: N Corregidas (blue border-left)

[Branch/Grouping Level]
  ├─ Branch Name (collapsible header, silver background)
  │
  └─ [Employee Card] (collapsible, white/zinc-900 background)
     ├─ [Collapsed: Summary Row]
     │  ├─ Name, {N} horas, {N} días, {N} problemas badge
     │  └─ [Expand/Collapse Chevron]
     │
     └─ [Expanded: Daily Rows]
        ├─ Date header row (light gray background)
        │
        ├─ Day detail row 1
        │  ├─ IN mark (type badge: green)
        │  ├─ OUT mark (type badge: orange)
        │  ├─ Duration (calculated)
        │  └─ Action button
        │
        ├─ Day detail row 2 (if time-off or no marks)
        │  ├─ "Día de descanso" or "Sin marcas registradas"
        │  └─ [Agregar marca] link (if admin permission)
        │
        └─ [Collapse] (chevron up button)

[Infinite Scroll Trigger]
  └─ Load next batch when user scrolls to 80% of visible list
```

### Expand/Collapse Behavior
- **Expand:** Smooth animation (framer-motion) of daily rows sliding down, background shifts slightly
- **Collapse:** Rows slide up, card returns to summary view
- **Persistence:** Expanded state saved to sessionStorage per employee ID
- **Default state:** All employees collapsed (load performance for 50+ employees)

### Status Badge Colors (from clockLogPresenter.ts)
- `valid`: `bg-green-100 text-green-800` (light), `dark:bg-green-900/20 dark:text-green-400` (dark)
- `anomaly`: `bg-amber-100 text-amber-800` (light), `dark:bg-amber-900/20 dark:text-amber-400` (dark)
- `orphan`: `bg-red-100 text-red-800` (light), `dark:bg-red-900/20 dark:text-red-400` (dark)
- `corrected`: `bg-blue-100 text-blue-800` (light), `dark:bg-blue-900/20 dark:text-blue-400` (dark)
- `pending`: `bg-gray-100 text-gray-800` (light), `dark:bg-gray-900/30 dark:text-gray-400` (dark)

### Sort Order
Employees in each branch are sorted by:
1. **Anomaly priority:** Employees with anomalies/orphaned marks at TOP (D-03)
2. **Within same status:** Alphabetical by last name

## Indicators & Feedback

### Problem Badges
- **Anomaly count badge:** Amber/orange pill badge with numeric count, positioned in employee card header
  - Shows only if count > 0
  - Example: `3 problemas` (orange bg, dark text)

- **Missing mark indicator:** Inline blue link text
  - Placement: In daily row where pair is incomplete
  - Copy: `Falta marca de {entrada|salida}. [Agregar marca]`
  - Trigger: Opens adjustment modal with pre-filled action type (ADD)

- **Orphaned mark indicator:** Orange inline alert in daily row
  - Copy: `Marca huérfana sin pareja. [Revisar]`
  - Trigger: Opens detail modal showing the lone IN or OUT mark

### Source Traceability
- **Visual icon next to timestamp:**
  - `⏱️` (clock icon) = Imported from device (`java_import` source)
  - `✋` (hand icon) = Manual entry (`manual` source)
  - `🔄` (refresh icon) = Adjusted entry (mark has `corrected` status)

- **Hover tooltip:** (on icon) "Importado desde reloj el 2024-04-01 10:23" or "Entrada manual agregada por Juan"

### Loading & Feedback
- **List loading:** Skeleton cards (8 dummy employee rows) while fetching marks
- **Mark detail modal loading:** Spinner in modal body, disabled confirm button
- **Pagination/Infinite scroll:** "Cargando más marcas..." indicator at bottom
- **Successful action:** Toast notification (green): "Marca actualizada correctamente"
- **Validation error:** Inline red text below form field + red border

## Responsive Design

### Breakpoints (Tailwind)
- **sm (640px):** Two-column card summary layout
- **md (768px):** Hide branch grouping on very small screens, collapse to employee-only view
- **lg (1024px):** Full three-level hierarchy visible
- **xl (1280px):** Wider card spacing

### Mobile Optimizations (< 640px)
- Collapse secondary info: Hide "Días trabajados", show only "N problemas" badge
- Date picker: Full-width input, larger touch target (44px min height)
- Action buttons: Full-width in cards, stacked vertically
- Status filter toggles: Horizontal scroll container (not wrapping)
- Employee summary row: Single-line layout with ellipsis overflow

## Component Specifications

### 1. Branch Header (NEW)
**Purpose:** Visual grouping of employees by Sucursal.

**Structure:**
```
┌──────────────────────────────────────────┐
│ 🏢 Sucursal: San José — 12 empleados     │
```

- Branch name (e.g., "Sucursal San José")
- Employee count in branch (e.g., "— 12 empleados")

### 2. Employee Card (Collapsed)
**Purpose:** Summary view of one employee's marks in a period.

**Structure:**
```
┌─────────────────────────────────────────────┐
│ Juan López Rodríguez          30 hrs | 3 ⚠️ │
│ 12 días trabajados                  [▼]    │
└─────────────────────────────────────────────┘
```

**Classes:** `bg-white dark:bg-zinc-900 rounded-lg border border-zinc-200 dark:border-zinc-700 px-4 py-3 cursor-pointer hover:shadow-md transition-shadow`

**Content:**
- **Left:** Employee name (bold, verde-primary color)
- **Middle:** Summary stats
  - Line 1: `{N} horas totales en período`
  - Line 2: `{N} días trabajados`
- **Right:** Anomaly badge (if problems > 0)
  - Badge: `{N} problemas` (amber/orange pill)
- **Far right:** Expand/Collapse chevron (rotates on toggle)

**Interaction:** Click anywhere on card to expand/collapse. Chevron click is same as card click.

### 3. Employee Card (Expanded)
**Purpose:** Show daily breakdown with IN/OUT marks for the employee.

**Structure:**
```
┌─────────────────────────────────────────────┐
│ Juan López Rodríguez          30 hrs | 3 ⚠️ │
│ 12 días trabajados                  [▲]    │
├─────────────────────────────────────────────┤
│                                             │
│ Lunes, 1 de abril de 2024                   │
│ ├─ 08:00 (IN)  ✓ Válida                    │
│ ├─ 17:00 (OUT) ✓ Válida — 9 horas         │
│ └─ [Corregir]                              │
│                                             │
│ Martes, 2 de abril de 2024                  │
│ ├─ 08:15 (IN)  ✓ Válida                    │
│ ├─ Falta marca de salida. [Agregar]        │
│                                             │
└─────────────────────────────────────────────┘
```

**Classes:** Same card container as collapsed, with animation to expand and show daily rows.

**Content per day:**
- **Date header:** Full date in Spanish (e.g., "Lunes, 1 de abril de 2024")
- **IN/OUT rows:** Timestamp + type badge + status badge + duration (for complete pairs)
- **Missing pair indicator:** Inline blue link text
- **Action button:** "Corregir" (orange) or "Ver detalles" (neutral) depending on status

### 4. Status Badge (Reuse Existing)
**Purpose:** Show mark status at a glance.

**Spec (from ClockLogStatusBadge.tsx):**
- Size: `text-xs` + `px-2.5 py-0.5`
- Shape: Fully rounded pill
- Colors: Per STATUS_COLORS in presenter (valid/pending/anomaly/orphan/corrected)
- Text: Per STATUS_LABELS in presenter (Spanish names)

**No changes needed.** Phase 34 reuses this component.

### 5. Adjustment Modal (Phase 34 OUT OF SCOPE)

- **Focus indicators:** Blue ring on focused interactive elements

- **Time format:** 24-hour (HH:MM), no AM/PM

## Implementation Constraints & Conventions

### Naming Conventions (from CLAUDE.md)
- **Frontend components:** PascalCase (e.g., `EmployeeBranchGroup.tsx`, `EmployeeClockCard.tsx`)
- **Frontend hooks:** camelCase with `use` prefix (e.g., `useEffectiveMarks.ts`)
- **Frontend services:** camelCase (e.g., `effectiveMarksService.ts`)
- **Imports:** Always use `@/` path alias, never `../../` beyond 1 level deep
- **Forms:** Use `react-hook-form` + `zodResolver` if adding new forms
- **API calls:** Always via `http.ts` service layer — never raw `fetch` in components
