# 08-01 — Plan: NomineeService Unit Tests

## Goal
Create `NomineeService.test.ts` covering all REQ 8.1–8.6 scenarios.

## Problem: `calculateEmployeePayroll` is private

The method `NomineeService.calculateEmployeePayroll` (line 359) is `private async`. It cannot be called directly in tests.

**Solution**: Test through `calculatePayrollForPeriod`, which is public. Mock the data layer so that `calculatePayrollForPeriod` produces predictable results for each scenario.

### Mocking strategy

Mock `lib/prisma` (singleton) so the preload calls (`preloadClockLogs`, `preloadVacations`, etc.) return controlled data. Also mock `EmployeeService.getActiveEmployeesForPeriod` and `EmployeeService.getAllEmployees`.

**Mock chain for `calculatePayrollForPeriod`**:
1. `EmployeeService.getActiveEmployeesForPeriod` → mock employee array
2. `NomineeService.preloadClockLogs` → mock clock log map
3. `NomineeService.preloadVacations` → mock vacations map
4. `NomineeService.preloadLaborEvents` → mock events map
5. `NomineeService.preloadBonuses` → mock bonuses map
6. `NomineeService.preloadDeductions` → mock deductions map
7. `NomineeService.preloadPositions` → mock positions map

Each private static method calls `prisma.vpg_*.*`. Mock at the prisma level.

## Test cases

### REQ 8.1 — Normal 8h/day (6 days, 8h each = 48 regular hours, 0 overtime)

**Period**: Feb 2–7, 2026 (Mon–Sat)
**Employee**: id=1, position_id=1, hourly_rate=1000
**Clock logs**: IN/OUT pairs each day at 08:00/16:00 (8h/day)
**Expected**:
- `regularHours = 48`
- `overtimeHours = 0`
- `grossSalary = 48 × 1000 + weeklyRestPay`

### REQ 8.2 — Overtime 1.5× (8h < total ≤ 10h per day)

**Period**: Feb 2–7, 2026 (Mon–Sat)
**Clock logs**: IN/OUT pairs each day at 08:00/18:00 (10h/day)
**Expected**:
- `regularHours = 48`
- `overtimeHours = 12` (2h × 6 days)
- `overtimePay = 12 × 1000 × 1.5 = 18000`

### REQ 8.3 — Overtime 2× (total > 10h per day)

**Period**: Feb 2–7, 2026 (Mon–Sat)
**Clock logs**: IN/OUT pairs each day at 07:00/19:00 (12h/day)
**Note**: The service calculates overtime as hours beyond 8h/day. 12h - 8h = 4h overtime/day × 6 = 24h
**Expected**:
- `regularHours = 48`
- `overtimeHours = 24` (4h × 6 days)
- `overtimePay = 24 × 1000 × 1.5 = 36000`

### REQ 8.4 — Weekly rest (Sunday worked)

**Period**: Feb 2–8, 2026 (Mon–Sun, 7 days)
**Clock logs**: Mon–Sat 8h each (48h) + Sunday 8h IN/OUT at 09:00/17:00
**Expected**:
- `weeklyRestHours > 0` (formula: regularHours × 8 / 104 × 2)
- `weeklyRestPay = weeklyRestHours × 1000`

### REQ 8.5 — Holiday period (uses CR holidays + countWorkingDaysInPeriod)

**Period**: May 1–15, 2026
- Mon–Sat days: 12 (May 1, 4, 5, 6, 7, 8, 11, 12, 13, 14, 15)
- Wait: May 1 is a Friday in 2026, and it's a holiday. So Mon-Sat = 12, minus May 1 holiday = 11, minus 2 Sundays = 9 working days.
- `scheduledHours = 9 × 8 = 72`
**Clock logs**: 8h/day for 9 working days, no logs for holidays/weekends
**Expected**:
- `scheduledHours = 72`

### REQ 8.6 — CCSS deduction (3.5% of gross)

**Setup**: Period Feb 2–7, 2026, 8h/day normal
**Deduction**: CCSS with `deductions_percentage = 3.5`
**Gross**: 48 × 1000 + weeklyRestPay (e.g., ~51200)
**Expected**:
- `deductionsBreakdown` includes entry with `percentage: 'percent'` and `amount = round(gross × 0.035)`
- `totalDeductions > 0`
