# 08-02 — Plan: Fix PayrollService.test.ts Mocks

## Goal
Fix the 2 failing tests in `PayrollService.test.ts` so `npm test` passes with 0 failures.

## Failing tests

### `getAllPayrolls` — "should retrieve all payrolls ordered by ID descending"
**Problem**: Mock doesn't include `include: { vpg_payroll_employee: true }` — Prisma will receive the mock with the include but the mock returns plain payroll arrays without the relation.

### `getAllPayrolls` — "should correctly map database fields to model fields"
**Problem**: Mock doesn't include `vpg_payroll_employee` relation and aggregated `total_*` fields that `PayrollService.getAllPayrolls` computes from the relation.

## Root cause

`PayrollService.getAllPayrolls` (line 39–85 of `PayrollService.ts`) does:
```typescript
const payrolls = await prisma.vpg_payrolls.findMany({
  include: { vpg_payroll_employee: true },  // <-- mock must include this
  orderBy: { payrolls_id: 'desc' }
});
// Then maps to include total_employees, total_gross, etc.
```

The mock at line 165 (`prismaMock.vpg_payrolls.findMany.mockResolvedValue(mockPayrolls)`) returns plain arrays without `vpg_payroll_employee` or the `total_*` computed fields.

## Fix

Add `vpg_payroll_employee: []` to each mock payroll record (empty array = 0 employees processed). The `total_*` fields will be computed as 0, which is fine for the existing tests since they don't assert on those fields.
