# Plan 05-03 — Audit Logs para Operaciones Críticas

**Phase:** 05 — Funcionalidad de Negocio Faltante  
**Task:** 5.4 — Audit log en operaciones críticas  
**Priority:** Should  

---

## Problem

`AuditLogsService` solo tiene métodos de lectura (`getAuditLogs`, `getAuditLogById`). La tabla `vpg_audit_logs` existe y está correctamente estructurada. No hay forma de escribir entradas de auditoría.

## Target Operations

Per REQUIREMENTS 5.4 — audit log en:
1. **Crear planilla** — `POST /api/payroll/create`
2. **Asignar deducción a empleado** — `POST /api/employee-deductions/assign`
3. **Cambiar estado de empleado** — `PUT /api/employee/:id` (cuando se actualiza el campo `status`)

---

## Steps

### 3.1 — Add `createAuditLog` to AuditLogsService

Add to `src/backend/src/service/AuditLogsService.ts`:

```typescript
static async createAuditLog(params: {
  userId: number;
  action: string;
  entity: string;
  entityId: number;
  details?: string;
}): Promise<void> {
  await prisma.vpg_audit_logs.create({
    data: {
      audit_logs_user_id: params.userId,
      audit_logs_action: params.action,
      audit_logs_entity: params.entity,
      audit_logs_entity_id: params.entityId,
      audit_logs_timestamp: new Date(),
      audit_logs_details: params.details || null,
    },
  });
}
```

### 3.2 — Audit log on create payroll

Edit `src/backend/src/controller/PayrollController.ts` — `createPayroll` method (line 48-49):

Add import at top:
```typescript
import { AuditLogsService } from "../service/AuditLogsService";
```

After `const payroll = await PayrollService.createPayroll(payrollData);` (line 48), before `res.status(201)` (line 49):
```typescript
await AuditLogsService.createAuditLog({
  userId: req.user.id,

  entity: 'payroll',

```

Note: `req.user.id` is available because `AuthMiddleware.verifyToken` runs before the controller (verified at PayrollRoutes.ts:10).
