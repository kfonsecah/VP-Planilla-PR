# Plan 05-02 — Implementar updateLastLogin con Migración Prisma

**Phase:** 05 — Funcionalidad de Negocio Faltante  
**Tasks:** 5.2 + 5.3  
**Priority:** Must  

---

## Problem

`vpg_users` no tiene campo `last_login`. `AuthService.updateLastLogin()` existe (línea 281 AuthService.ts) pero solo hace `console.log` — no persiste nada a la DB.

## Steps

### 2.1 — Add `last_login` to schema

Edit `src/backend/prisma/schema.prisma` model `vpg_users` (line 291):

Add after `user_version`:
```prisma
  user_last_login   DateTime?  @db.Timestamp(6)
```

### 2.2 — Generate Prisma migration

```bash
cd src/backend
npx prisma migrate dev --name add_last_login_to_users
```

### 2.3 — Implement AuthService.updateLastLogin

Edit `src/backend/src/service/AuthService.ts` method `updateLastLogin` (lines 281-287):

Replace the stub body with:
```typescript
static async updateLastLogin(userId: number): Promise<void> {
  await prisma.vpg_users.update({
    where: { user_id: userId },
    data: { user_last_login: new Date() },
  });
}
```

Note: The call to `AuthService.updateLastLogin(result.user.id)` already exists in `AuthController.ts:76`. No changes needed there.

### 2.4 — TypeScript check

```bash
cd src/backend && npx tsc --noEmit
```

## Verification

```bash
# Migration file exists
ls src/backend/prisma/migrations/ | grep add_last_login

# TypeScript passes
cd src/backend && npx tsc --noEmit

# UpdateLastLogin now uses prisma (no console.log)
grep -n "console.log.*login" src/backend/src/service/AuthService.ts  # should return nothing
```
