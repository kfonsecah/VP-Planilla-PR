  <behavior>
    - detectOrphans(sessionId): Finds all IN logs for the session's employee/date range that have no matching OUT within 24h → updates status to 'orphan'. Returns count of orphans found.
    - detectDoubleEntry(sessionId): Finds pairs of consecutive IN logs for same employee without OUT between them → updates both to status 'anomaly'. Returns count.
    - detectDoubleExit(sessionId): Finds pairs of consecutive OUT logs for same employee without IN between them → updates both to status 'anomaly'. Returns count.
    - detectLongSessions(sessionId): Finds IN→OUT pairs where duration > 16 hours → marks as status 'anomaly'. Returns count.
    - runPostImportAnalysis(sessionId): Orchestrates all detection methods in sequence (orphans → double entry → double exit → long sessions). Returns { orphans, doubleEntry, doubleExit, longSessions, total }.
    - All methods are static. All methods operate only on logs belonging to the given session (filter by clock_logs_import_session_id).
    - Detection only processes logs with status = 'pending' (not already corrected or anomalous).
    - After marking anomalies/orphans, remaining pending logs are marked as 'valid'.
  </behavior>
  <action>
    Create src/backend/src/service/ClockLogAnalysisService.ts as a static service class following the project pattern (static methods only, no instantiation).

    Import: `import { prisma } from '../lib/prisma';`

    Implement these static methods:

    1. `static async detectOrphans(sessionId: number): Promise<number>`
       - Query all logs for this session: `prisma.vpg_clock_logs.findMany({ where: { clock_logs_import_session_id: sessionId, clock_logs_status: 'pending' }, orderBy: { clock_logs_timestamp: 'asc' } })`
       - Group by employee_id
       - For each employee, iterate chronological logs. An IN is orphan if no OUT follows within 24 hours (timestamp + 24h). An OUT is orphan if no preceding IN exists within 24h.
       - Use `prisma.vpg_clock_logs.updateMany({ where: { clock_logs_id: { in: orphanIds } }, data: { clock_logs_status: 'orphan' } })`
       - Return count of orphans marked

    2. `static async detectDoubleEntry(sessionId: number): Promise<number>`
       - Query logs for session, group by employee, sort by timestamp
       - Find consecutive IN/IN pairs (two IN logs with no OUT between them for same employee)
       - Mark both logs in each pair as status = 'anomaly'
       - Return count of anomalies marked

    3. `static async detectDoubleExit(sessionId: number): Promise<number>`
       - Same pattern as double entry but for OUT/OUT consecutive pairs
       - Mark both logs as status = 'anomaly'
       - Return count

    4. `static async detectLongSessions(sessionId: number): Promise<number>`
       - For each employee, find IN→OUT pairs within the session
       - Calculate duration in hours: `(OUT.timestamp - IN.timestamp) / (1000 * 60 * 60)`
       - If duration > 16 hours, mark both IN and OUT as status = 'anomaly'
       - Return count

    5. `static async runPostImportAnalysis(sessionId: number): Promise<{ orphans: number; doubleEntry: number; doubleExit: number; longSessions: number; total: number }>`
       - Call detectOrphans, detectDoubleEntry, detectDoubleExit, detectLongSessions in sequence
       - After all detection, mark remaining 'pending' logs for this session as 'valid': `prisma.vpg_clock_logs.updateMany({ where: { clock_logs_import_session_id: sessionId, clock_logs_status: 'pending' }, data: { clock_logs_status: 'valid' } })`
       - Return aggregated counts

    6. `static async markValid(sessionId: number): Promise<number>`
       - Mark all remaining pending logs for the session as valid
       - Return count of logs marked

    Each method must have full JSDoc with @param, @returns, @throws.
