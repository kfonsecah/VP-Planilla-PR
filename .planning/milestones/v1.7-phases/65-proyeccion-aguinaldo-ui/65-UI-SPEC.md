## Component Specifications

### 1. Aguinaldo Card (Employee Profile)

**Location:** Employee profile page, dedicated "Aguinaldo acumulado" section

**Structure:**
- Container: bg-white dark:bg-zinc-900, border border-zinc-200 dark:border-zinc-800, rounded-lg, p-md (16px), shadow-sm
- Left border accent: 4px border-l-verde-primary (#4A5D3A)

**Layout (flex column):**
1. **Header row (flex between):**
   - Left: "Aguinaldo acumulado" (font-bold, 16px, verde-primary)
   - Right: "Acumulado a [DD/MM/YYYY]" (caption, 12px, text-zinc-500, regular weight)

2. **Amount display:**
   - Large amount: "₡[cantidad]" (20px, bold, #1F2937)
   - Format: 2 decimals, right-aligned within card
   - Example: "₡1,234,567.89"

3. **Progress bar:**
   - Full width, height 6px, bg-zinc-200 dark:bg-zinc-700, rounded-full
   - Filled portion: bg-verde-primary, width = %(months completed / 12)
   - Tooltip on hover: "[N] meses completados del período (1° dic - 30 nov)"

4. **Metadata row (grid 2 cols, gap-lg):**
   - Col 1: 
     - Label: "Basado en" (caption, 12px, zinc-500, regular weight)
     - Value: "[N] planillas aprobadas" (14px, zinc-800, regular weight)
   - Col 2:
     - Label: "Proyección" (caption, 12px, zinc-500, regular weight)
     - Value: "₡[proyectado al 20 dic]" (14px, verde-secondary, regular weight)

**States:**
- Default: as above
- Loading: skeleton placeholder (gray pulse animation, same dimensions)
- Empty (no payrolls): "₡0.00" with caption "Sin planillas aprobadas en el período"
- Error: show cached value with small error icon + tooltip "Verificando últimos datos"

## Interaction Specs

### Aguinaldo Card

- **Progress bar hover:** Show tooltip with exact month count and date range
- **Amount click:** No action (read-only)
- **Card refresh:** If user navigates away and back to profile, refetch from `GET /employees/:id/aguinaldo` via http.ts service
- **Loading state:** Show Skeleton (gray pulse, same card dimensions) while fetching

### Wizard Step 3 Table

- **Cell hover:** Show tooltip preview of contribution that will be added
- **Cell data:** Read-only (fetched from `GET /payroll/:id/aguinaldo-summary`)
- **Sort:** Aguinaldo column is sortable ascending/descending like other columns

### Wizard Step 4 Summary

- **Visibility:** Display only if `GET /payroll/:id/aguinaldo-summary` returns successfully with at least one employee
- **Updates:** Re-render when employee selection changes in Step 3
- **Animation:** Fade in with 200ms easing on mount

## Dark Mode

All color values have explicit dark variants:

- `bg-white` → `dark:bg-zinc-900`
- `text-zinc-800` → `dark:text-white`
- `text-zinc-600` → `dark:text-zinc-300`
- `border-zinc-200` → `dark:border-zinc-800`
- `bg-verde-card` → (use verde-card value directly — no dark override needed for accent colors)
- `bg-zinc-200` (progress track) → `dark:bg-zinc-700`

All components must include `dark:` variants in className strings.

## API Integration Points

**Frontend calls backend via `@/services/http.ts` ONLY:**

1. `GET /employees/:id/aguinaldo`
   - Returns: `AguinaldoAccrual` object
   - Called on: Employee profile load
   - Cache: 5-minute client-side cache (optional, handled by hook)

2. `GET /payroll/:id/aguinaldo-summary`
   - Returns: Array of `{ employeeId, employeeName, accruedBeforeThisPayroll, thisPayrollContribution, totalAccruedWithThis }`
   - Called on: Wizard step 3 load OR when employee selection changes
   - Cache: Invalidate on step back/forward

## Responsive Behavior

- **Mobile (< 640px):** 
  - Card width: 100% (full-width with padding)
  - Summary grid: Stack to 1 column
  - Table: Horizontal scroll with sticky left column

- **Tablet (640px - 1024px):**
  - Card width: 100% or constrained to 500px in grid layout
  - Summary grid: 2 columns
  - Table: Normal layout

- **Desktop (> 1024px):**
  - Card width: fits naturally in 3-column grid (if on dashboard)
  - Summary grid: 2 columns
  - Table: Normal layout
