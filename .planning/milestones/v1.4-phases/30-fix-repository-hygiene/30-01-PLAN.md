<objective>
Fix repository hygiene issues by purging tracked-but-ignored files and enforcing a consistent lock file policy for deterministic builds.
</objective>

<execution_context>
The v1.4 audit identified that several files (like .DS_Store, .vscode/settings.json, etc.) are currently tracked by git even though they match .gitignore patterns. Additionally, the current policy of ignoring package-lock.json files is inconsistent with industry standards for application reproducibility.
</execution_context>

<tasks>

<task type="auto">
  <name>Task 1: Purge tracked but ignored files from git cache</name>
  <action>
Run the following command to remove specifically identified tracked-but-ignored files from the index (without deleting them from disk):

```powershell
git rm --cached .vscode/settings.json
git rm --cached src/.DS_Store
git rm --cached src/Java/clocklogs/.vscode/settings.json
git rm --cached src/Java/clocklogs/dependency-reduced-pom.xml
git rm --cached src/backend/.DS_Store
git rm --cached src/frontend/parse_tmp.js
```

Also check for any tracked .pyc files identified in audit:
```powershell
git ls-files "*.pyc" | % { git rm --cached $_ }
```
  </action>
  <acceptance_criteria>
    - Command `git ls-files -i -c --exclude-standard` returns no output for the identified files.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Update .gitignore policy for lock files</name>
  <files>.gitignore</files>
  <action>
Modify .gitignore to allow tracking of package-lock.json in core application directories while keeping general ignores for others if needed.

Change:
```gitignore
# Package manager lock files (regenerable from package.json)
**/package-lock.json
```

To:
```gitignore
# Package manager lock files (tracked in apps for deterministic builds)
# **/package-lock.json (removed general ignore)
!src/backend/package-lock.json
!src/frontend/package-lock.json
```
  </action>
  <acceptance_criteria>
    - .gitignore no longer has a blanket ignore for package-lock.json.
    - .gitignore explicitly unignores package-lock.json in backend and frontend.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 3: Verify hygiene and stage lock files</name>
  <action>
Check status and stage the newly unignored lock files to establish baseline hygiene.

```powershell
git status
# If package-lock.json files appear as untracked, they should be added in a future commit (not now per mandates)
```
  </action>
  <acceptance_criteria>
    - git status shows the files removed in Task 1 as "deleted" (from index).
    - .gitignore changes are staged.
  </acceptance_criteria>
</task>

</tasks>
