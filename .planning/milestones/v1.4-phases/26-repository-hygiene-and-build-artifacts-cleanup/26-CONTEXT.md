# Phase 26: Repository Hygiene and Build Artifacts Cleanup - Context

**Gathered:** 2026-04-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove already-tracked build artifacts from git and reinforce `.gitignore` for the multi-stack repo (Node backend, Next.js frontend, Java utility) so generated outputs are never accidentally re-committed. Does not include changes to build scripts, CI/CD pipelines, or deployment workflows.

</domain>

<decisions>
## Implementation Decisions

### Artifact removal scope
- Remove `src/backend/dist/` (compiled TypeScript JS output) from git tracking
- Remove `src/Java/clocklogs/target/` (`.class` files, `.jar` files, Maven metadata) from git tracking
- Audit `src/frontend/.next/` to confirm it is already excluded — add explicit rule if not

### Untracking method
- Use `git rm --cached -r` to remove artifacts from the git index without deleting local files
- No history rewrite (`git filter-repo` not needed) — artifacts are not sensitive data
- Files remain on disk after untracking so local builds continue to work

### .gitignore structure
- Single root-level `.gitignore` file (do not create per-package files)
- Organize with clear sections per stack: Backend (Node/TypeScript), Frontend (Next.js), Java utility, General

### *.json global ignore fix
- Remove the global `*.json` ignore rule — it silently hides any new JSON file outside `.claude/` and `.planning/`
- Remove the associated `!.claude/**/*.json` and `!.planning/**/*.json` re-include hacks (no longer needed)
- Add only specific JSON ignores where actually needed (e.g., local secrets files if any)

### Claude's Discretion
- Exact order and grouping of sections within the new `.gitignore`
- Whether to add `node_modules/` explicitly if already absent from tracking
- Commit message format for the cleanup commit

</decisions>

<specifics>
## Specific Ideas

- Build locally when needed — the repo does not need dist/ or target/ tracked for any deployment reason
- The cleanup should result in a verifiably smaller `git status` with no unintended untracked files appearing
