Do NOT use `http.get()` — it unwraps the data field and loses pagination metadata. Always use `http.raw()`.

Create `src/frontend/src/hooks/useEffectiveMarks.ts` following the pattern of `useClockLogs.ts`.

The hook manages: filters, paginated data with infinite scroll append, loading/error states, import sessions.

**State shape:**
- `data: EffectiveClockLog[]` — accumulated across pages (append on loadMore)
- `totalCount: number`
- `page: number` — current page loaded
- `hasMore: boolean` — derived: page * pageSize < totalCount
- `isLoading: boolean` — true during initial load and filter changes
- `isLoadingMore: boolean` — true only during loadMore() calls
- `error: string | null`
- `filters: EffectiveMarksFilters` — initDate/endDate default to current biweekly period (first half if day <= 15, else second half)
- `importSessions: ImportSession[]` — from ClockLogsService.getImportSessions(5)

**Helper functions (module-level, not inside hook):**
```typescript
const PAGE_SIZE = 20;

function getDefaultDates(): { initDate: string; endDate: string } {
  const now = new Date();
  const pad = (n: number) => String(n).padStart(2, '0');
  const year = now.getFullYear();
  const month = now.getMonth() + 1;
  const day = now.getDate();
  if (day <= 15) {
    return {
      initDate: `${year}-${pad(month)}-01`,
      endDate: `${year}-${pad(month)}-15`,
    };
  }
  const lastDay = new Date(year, month, 0).getDate();
  return {
    initDate: `${year}-${pad(month)}-16`,
    endDate: `${year}-${pad(month)}-${pad(lastDay)}`,
  };
}
```

**`applyDatePreset` (per D-01):**
Handles 'first_half' (1–15 of current month), 'second_half' (16–last day), 'this_month' (1st to today).
On preset apply: reset data to [], reset page to 1, update filters.

**`setFilters` (partial update):**
On any filter change: reset data to [], reset page to 1, then re-fetch.

**`loadMore` (infinite scroll per D-05):**
- If `isLoadingMore` or !hasMore: return early
- Fetch next page (currentPage + 1), APPEND results to data[]
- Increment page state
- Do NOT reset data[]

**`refresh`:**
Reset data to [], reset page to 1, fetch page 1 with current filters.

**Import sessions:**
Fetch once on mount via `ClockLogsService.getImportSessions(5)`. Import `ClockLogsService` and `ImportSession` from `@/services/clockLogsService`. Store in `importSessions` state.

**Return shape (exactly):**
```typescript
return {
  data,
  totalCount,
  page,
  hasMore,
  isLoading,
  isLoadingMore,
  error,
  filters,
  importSessions,
  setFilters,
  applyDatePreset,
  loadMore,
  refresh,
};
```

**useEffect triggers:**
- Filter changes (initDate, endDate, branch_id, employee_id, status array reference): reset and fetch page 1
- Mount: fetch import sessions once

**Important:** `useCallback` on all async functions. Use functional state updates (setData(prev => [...prev, ...newItems])) for loadMore append.
