<task type="auto">
  <name>Task 1: Update schema.prisma with new models and enums</name>
  <files>src/backend/prisma/schema.prisma</files>
  <action>
    Modify the Prisma schema to include:

    1. Update `ClockLogSource` enum: Add `device` value (keep existing java_import, excel_import, manual).

    2. Add enums:
       - `ClockLogAdjustmentType`: `ADD`, `EDIT`, `VOID`
         - ADD: marca faltante agregada manualmente (no existe clock_log original)
         - EDIT: corrección de timestamp de marca existente
         - VOID: anulación de marca errónea (soft delete)
       - `ClockLogAdjustmentStatus`: `ACTIVE`, `INACTIVE`
       - `PayrollStatus`: `BORRADOR`, `APROBADA`, `PAGADA`

    3. Add `vpg_clock_log_adjustments` model:
       - `adjustment_id` (PK, autoinc)
       - `adjustment_clock_log_id` (Int? — nullable FK to vpg_clock_logs; null when type=ADD)
       - `adjustment_employee_id` (Int — required FK to vpg_employees; always present)
       - `adjustment_type` (ClockLogAdjustmentType)
       - `adjustment_original_timestamp` (DateTime? @db.Timestamp(6) — valor original; null para ADD)
       - `adjustment_new_timestamp` (DateTime? @db.Timestamp(6) — valor corregido; null para VOID)
       - `adjustment_log_type` (ClockLogType — IN o OUT; siempre requerido)
       - `adjustment_justification` (String @db.Text — mínimo 10 chars, enforced in Zod)
       - `adjustment_status` (ClockLogAdjustmentStatus @default(ACTIVE))
       - `adjustment_created_by` (Int — FK to vpg_users)
       - `adjustment_created_at` (DateTime @default(now()) @db.Timestamp(6))
       - `adjustment_version` (Int @default(1) — optimistic locking)
       - Relations: vpg_clock_logs? (optional), vpg_employees, vpg_users
       - Index: `[adjustment_clock_log_id, adjustment_status]`
       - Index: `[adjustment_employee_id]`

    4. Add `vpg_payroll_recalculations` model:
       - `recalc_id` (PK, autoinc)
       - `recalc_payroll_id` (FK to vpg_payrolls)
       - `recalc_reason` (String @db.Text)
       - `recalc_timestamp` (DateTime @default(now()) @db.Timestamp(6))
       - `recalc_created_by` (FK to vpg_users)
       - `recalc_data_snapshot` (Json — snapshot of vpg_payroll_employee before recalc)
       - Index: `[recalc_payroll_id]`

    5. Update `vpg_payrolls`:
       - Change `payrolls_status` from `String @db.VarChar(20)` to `PayrollStatus` enum
       - Add `payrolls_approved_by` (Int? — nullable FK to vpg_users)
       - Add `payrolls_approved_at` (DateTime? @db.Timestamp(6))
       - Add `payrolls_notes` (String? — texto libre del jefe)
       - Add `payrolls_reopened_at` (DateTime? @db.Timestamp(6) — audit trail de reapertura)
       - Add `payrolls_reopen_reason` (String? — razón de reapertura)
       - Add relation: vpg_payroll_recalculations[]
       - Add relation: vpg_users? (for approved_by)

    Ensure naming follows the `{table}_{column}` pattern (e.g., `adjustment_id`, `recalc_id`).
  </action>
