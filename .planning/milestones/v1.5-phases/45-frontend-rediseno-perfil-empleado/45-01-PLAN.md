# Phase 45, Plan 01: Enrutamiento y Base del Perfil (Tabs)

## Objetivo
Crear la estructura base de la página `app/pages/employee/[id]/page.tsx` que alojará el nuevo perfil consolidado con navegación por Tabs, aislado del resto de la tabla de empleados.

## Tareas

1. **Crear ruta dinámica**:
   - Crear el directorio `src/frontend/src/app/pages/employee/[id]`.
   - Crear `page.tsx` dentro de esa ruta.

2. **Fetching Base (Service + Hook)**:
   - Crear o ampliar el servicio para cargar un empleado detallado por su `id` (`employeeService.getEmployeeById`).
   - Crear `useEmployeeProfile` hook para manejar el estado de carga y tener la data del empleado accesible.

3. **Componente ProfileSkeleton & Layout**:
   - Construir un Header para mostrar Foto/Avatar + Nombre y Puesto del empleado.
   - Reutilizar o adaptar el patrón `EmployeeTabs` para crear `EmployeeProfileTabs.tsx` que controle las áreas del perfil (Resumen, Planillas, Eventos, Docs).

4. **Regreso al listado**:
   - Integrar un botón "Volver a Empleados" que lleve a `/pages/employee/list`.

## Verificación
- Visitar `/pages/employee/1` y confirmar que renderiza la estructura con Tabs sin colapsar.
- Confirmar que al estar cargando muestra skeleton.
- Confirmar botón para regresar a listado.
