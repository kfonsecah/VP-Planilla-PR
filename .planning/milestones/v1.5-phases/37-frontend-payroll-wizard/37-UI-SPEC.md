## Interaction Contracts

### Wizard Navigation
- **Progression:** Step 1 → Step 2 → Step 3 (forward only via primary CTA)
- **Back navigation:** Explicit "Cambiar período" button in Step 2 returns to Step 1
- **State persistence:** All wizard state in usePayrollWizard hook — data persists between steps

### Step 1: Period Selection
- Display period cards in 2-column grid (mobile: 1 column)
- Each card shows: icon, label (e.g., "1ª Quincena Abril 2026"), date range, "Actual" badge if current
- Selected card: green border (#16a34a), green subtle background
- On select → auto-calculate → navigate to Step 2

### Step 2: Review Calculation
- Header with period label and "Cambiar período" back link
- Warning banner (yellow) if any employee has inconsistencies
- Reuse existing PayrollResults component with expandible employee rows
- Primary CTA: "Revisar y continuar" → navigate to Step 3
- Auto-create payroll record in BORRADOR state via Phase 36 API

### Step 3: Executive Summary + Confirmation
- Summary card: Period, Employee count, Total Bruto, Total Neto
- Primary CTA: "Aprobar Planilla" → opens ConfirmDialog
- ConfirmDialog requires user to type "APROBAR" (UX-02 requirement)
- On confirm → call Phase 36 `/payroll/{id}/approve` endpoint → transition to APROBADA
- Success toast: "Planilla aprobada exitosamente"

### State Machine Integration
| Action | Endpoint | Transition |
|--------|----------|------------|
| Calcular (auto) | POST /payrolls | BORRADOR |
| Aprobar | POST `/payroll/{id}/approve` | BORRADOR → APROBADA |
| Marcar como pagada | POST `/payroll/{id}/pay` | APROBADA → PAGADA |
