## Modal & Dialog Specifications

### Import File Modal
- **Type:** FormModal pattern (fullscreen or max-w-2xl)
- **Animation:** backdropVariants + modalVariants (spring damping: 20, stiffness: 250)
- **Title:** "Cargar Marcas de Reloj"
- **Subtitle:** "Seleccione un archivo Excel. El sistema detectará automáticamente las columnas."
- **Close button:** XMarkIcon (top-right), text-white/80 on dark background
- **Footer actions:** "Cancelar" (secondary) + "Analizar y Detectar" (primary blue)
- **File input:** Accept `.xlsx, .xls, .csv` — show file size validation (max 10MB)

### Import Preview Table
- **Columns (auto-detected, displayed for confirmation):**
  - Empleado (resolved name or alias)
  - Fecha (formatted DD/MM/YYYY)
  - Hora (formatted HH:MM)
  - Tipo Inferido (IN / OUT / Dudoso)
  - Confianza (visual badge: verde/amarillo/rojo)
- **Row styling:** Hover highlight (bg-zinc-50), selected row bold
- **Sorting:** Click column header to sort (for spot-checking)
- **Pagination/scroll:** If >100 rows, show scroll container with sticky header

### Audit View (Main Page)
- **Layout:**
  - **Sidebar (left):** Filters (date range, "Solo con problemas" toggle), stat cards (empleados procesados, días revisados, marcas dudosas)
  - **Main content (right):** Hierarchical list of employees (expand/collapse)
- **Employee card (collapsed):**
  - Empleado name (bold)
  - Días con marcas: N
  - Estado: X revisados / Y totales (badge showing progress)
  - Expand arrow
- **Employee expanded → Day rows:**
  - Date (DD/MM/YYYY)
  - Hours calculated: HH.MM (e.g., 8.50h)
  - Confidence badge (average or "mixed" if marks vary)
  - Action: "Confirmar Día" button (toggles to "Revisado" state)
  - Expand arrow for marks
- **Day expanded → Mark rows:**
  - Time (HH:MM)
  - Type (IN / OUT / Dudoso)
  - Confianza badge (verde / amarillo / rojo)
  - Action: "Cambiar Tipo" (inline select dropdown)
  - Hover action: "Editar" (pencil icon, opens EditClockLogModal)

## Interaction & State Contract

### File Import Flow

**Step 1: Upload**
- Input: `.xlsx` / `.xls` / `.csv` file (max 10MB)
- Validation: File size, extension
- Error: Toast alert if invalid
- Success: Auto-parse trigger

**Step 2: Parse & Preview**
- Action: System auto-detects columns (name, date, time) + infers IN/OUT by time
- Display: Table with detected values (Empleado, Fecha, Hora, Tipo, Confianza)
- User can: Review, sort columns, scroll; approve or discard
- Error: If <1 valid row detected → show error message + allow re-upload

**Step 3: Confirm Import**
- Action: User clicks "Confirmar Importación" button
- Validation: At least 1 row must be valid
- Success: Data saved to backend; show audit view
- Failure: Show error detail per row, allow edit/retry

### Audit View Interaction

**Initial State:**
- All employees listed (collapsed)
- Stat cards show summary (X empleados, Y días, Z dudosas)
- "Solo con problemas" toggle is OFF

**User Action: Toggle Filter**
- Filter ON: Show only employees/days with unconfirmed or red-confidence marks
- Filter OFF: Show all

**User Action: Expand Employee**
- Collapse/expand days for that employee
- Show "X revisados / Y totales" progress

**User Action: Expand Day**
- Show all marks (time, type, confidence)
- Show "Horas calculadas: H.MM"
- Show "Confirmar Día" button

**User Action: Confirm Day**
- Button state: OFF → ON ("Revisado" badge)
- Visual feedback: Day row highlighted in green or grayed out
- Backend call: POST to mark-as-reviewed endpoint (or field in adjustments table)
- No other action required for that day

**User Action: Change Mark Type Inline**
- Click "Cambiar Tipo" → Dropdown (IN / OUT)
- Select new type
- Modal opens: EditClockLogModal (existing pattern)
- User enters justification (required, min 10 chars)
- Success: Mark type updated, row re-rendered with new type

**User Action: Edit Mark Time**
- Click "Editar" pencil icon → Opens EditClockLogModal
- Modal allows: time adjustment + justification
- Success: Backend saves EDIT adjustment; row updates

## Validation & Error Handling

### Form Validation

**File Upload:**
- File size: 1–10 MB
- File type: .xlsx, .xls, .csv only
- Required: At least 1 valid row after parsing

**Justification Text (for EDIT/VOID):**
- Required for all manual adjustments
- Minimum: 10 characters
- Real-time validation: Show "X/10 caracteres mínimo" hint below textarea

**Time Input:**
- Format: HH:MM (24-hour)
- Validation: 00:00–23:59

### Error Messages (Spanish, business language)

| Scenario | Message |
|---|---|
| File too large | "El archivo es demasiado grande (máx. 10 MB)" |
| File type invalid | "Formato no soportado. Use Excel (.xlsx/.xls) o CSV (.csv)" |
| No valid rows | "No se detectaron empleados válidos en el archivo. Revise los nombres y formatos de fecha/hora." |
| Employee not found (alias) | "El empleado '{alias}' no existe en el sistema." |
| Time outside all windows | "Marca con confianza baja: la hora está fuera de las ventanas configuradas. ¿Desea confirmar manualmente?" |
| Empty justification | "Justificación requerida (mínimo 10 caracteres)" |
| Parse error (Excel) | "Error al leer el archivo. ¿Es un archivo Excel válido?" |
